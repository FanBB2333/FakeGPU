What Was Patched

New Stubs Added

  • torch._C._cuda_clearCublasWorkspaces — Lightning Fabric calls this during CUDA memory cleanup
  • transformers.utils.is_torch_cuda_available — LLaMA-Factory reads this to decide CUDA vs CPU path
  • transformers.utils.is_torch_bf16_gpu_available — LLaMA-Factory reads this for dtype selection
  • accelerate.utils.imports CUDA detection — patches is_cuda_available for robust accelerate integration
  • torch.distributed._get_default_group & new_group — fallback stubs for distributed group management

Matmul Precision Fix

Torch 2.9+ introduced per-backend matmul precision tracking. Setting backends.cuda.matmul.allow_tf32 (new API) then calling torch.get_float32_matmul_precision() (legacy API) throws RuntimeError.

FakeGPU now intercepts both get/set_float32_matmul_precision and manages precision state in Python, bypassing the C++ backend entirely. This fixes cross-test contamination between HF Trainer (tf32=True) and Lightning Fabric.

Pytest Configuration

Added [tool.pytest.ini_options] to pyproject.toml with testpaths and pythonpath pointing to test/. This ensures hf_test_utils is importable on all platforms (previously worked by accident on macOS).

Known Limitations

  • bitsandbytes / QLoRA — requires real CUDA kernels
  • Flash Attention — requires CUDA kernel
  • DeepSpeed ZeRO — requires CUDA kernel
  • cudaLaunchKernel is a no-op; no actual GPU computation
  • Stream/Event are API stubs only (no real async)
